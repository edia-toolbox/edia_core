    eParam

    Parameter package definition to send along with an event.

    EventManager

    EventManager handles eventlisteners and fire events when triggered.
    It uses eparam class to be able to send custom packages along with the event.

    Core manager of the experiment. Singleton so available throughout the system.

    Main container to store sessions config, either from disk, editor or network

    Tuple of strings. Serializable in the inspector and dictionaries are not

    ExperimentManager.TrialSequenceValues

    Stringlist with trial setting values

    ExperimentManager.TrialSequenceValuesContainer

    Container for (de)serializing a list to JSON

    FileManager

    Static class to handle file IO

    InputActionHandler

    Mapping new Input system InputAction to eDIA EventManager

    MainMenu

    Main menu can be opened in the app with the action 'Menu'
    It generates a list of scenes (which are tasks) and the default 'continue' and 'quit' option

    MainMenu.aScene

    A scene menuitem container

    NetworkComManager

    Communication manager interface. Translates internal commands into network packages and viseversa

    NetworkManager

    Manages network connections, disconnections, auto connect, etc

    ScreenFader

    Handles fading the camera view

    SystemManager

    The SystemManager is a Singleton and therefor reachable from whatever script.
    The script travels along the scenes that are loaded as it is DontDestroyOnLoad.
    Responsible for loading/unloading, user related actions, top level application stuff.
    Has references to the XR rig camera and hands for the rest of the application.

    TaskManagerBase

    Link between ExperimentManager and the custom user task.
    Hidden base where the custom task template is build upon.

    Each task has it's own settings.
    Container to deserialize the TaskSettings.json file in

    TaskManagerBase.TrialStep
